## Суть в одном абзаце

Нетехнический founder за 6 месяцев собрал B2C web app, сам снял UGC на TikTok/Reels/Shorts (3 поста/день), вышел на стратегическую продажу Jenni AI. Главный тезис: **покупают не продукт, а дистрибуцию и навык виральности**; технический moat от AI ослаб, асимметрия сместилась в контент и формат.

## Ключевые идеи

### 1. Moat = дистрибуция, не код

- Примеры: MyFitnessPal → Cal AI, Quizlet → Coconote / Answers AI — общее: **формат + охват**, не уникальный код.
- «Средний продукт + отличная дистрибуция» бьёт «отличный продукт + слабая дистрибуция».
- Контент — learnable skill (автор не постил TikTok до старта; первый ролик — 200k views).

### 2. Сначала формат, потом продукт

Классика «problem → validate → build → market» — **наоборот и мёртва**.

**Критерии формата (все три):**

| Критерий | Смысл |
|----------|--------|
| Viral | Уже залетает на TikTok сейчас, не «может» |
| Scalable | 50+ вариаций без выгорания |
| Sustainable | Пережил хотя бы один цикл усталости |

**Где искать формат:**

- **Path A:** приложения с выручкой → разбор их winning content (Halo AI prank, UMax face rating).
- **Path B:** ecom / info / dropshipping — формат есть, продукт слабый → **взять формат, сделать app с нормальной LTV и retention**.

Совет: скроллить **TikTok**, не Reels (Reels ~2 недели отстаёт).

**MVP:** work backwards от формата; Cursor/Claude Code; **≤2 недель** до live; не строить фичи, которых нет в ролике.

**Ловушка:** one-and-done app + viral format = churn machine. Нужен продукт с **привычкой** (daily/weekly).

### 3. Первые 100 видео — только сам

- Без агентов, AI-инфлюенсеров, автоматизации, найма креаторов на старте.
- Причины: понять «что хорошо», бесплатно vs Meta ads, **viral sense** копится и — причина exit.
- Практика: **3×/день** на TikTok + Reels + Shorts; несколько аккаунтов; смотреть на **средний trending view**, не только на случайные хиты.

### 4. Production system (<$100 за 6 месяцев)

- **50 hook clips**, demo max **3 uses** (perceptual hashing / shadowban при переиспользовании).
- Hooks: Fiverr/Upwork <$2 или друзья/семья.
- **Batch:** ~16 ч в воскресенье — съёмка демо; в неделю только постинг.
- **Портфель контента:** 50% проверенные форматы, 25% итерации, 25% moonshots.
- Max **3 поста/день/аккаунт** на TikTok; равномерно по дню; наращивать объём постепенно.

**Тренировка viral sense:** 30 мин ниша + 30 мин вне ниши («дозволенный doomscroll»).

### 5. Organic vs paid

- Paid умирает, когда перестаёшь платить; organic — **долгий хвост** (CiteSure рос год без новых постов).
- Видео ≠ 24h lifespan; могут взлететь через недели/месяцы.
- Due diligence Jenni: **>80% net margin**, CAC ~$0.

### 6. Retention = multiple при exit

- Высокий churn → MRR «не recurring» → **ниже мультипликатор**.
- Советы: сильный продукт; vitamin → painkiller в копи; value в cancellation flow; retention offers; понимать психологию пользователя.

### 7. Как искать покупателя

1. Marketplaces (Acquire, TrustMRR, Flippa, Microns) — осторожно с TTM для молодых apps.
2. Brokers (Empire Flippers) — % от сделки.
3. Outbound — ICP, список, cold.
4. **Inbound (лучший)** — viral app сам попадает в поле зрения strategics.

Кейс Matt: Jenni написали в DM на «креаторский» аккаунт → partnership → acquisition. Inbound через контент.

### 8. Mindset

- Неприятная работа (500 постов, 16h съёмка, месяц <1k views) — **и есть актив**.
- Не срезать через AI/автоматизацию на этапе обучения.
- «Owning an app feels like wealth. Owning an exit is wealth.»

## План применения (для speaker-fix / app-ad каруселей)

### Фаза 0 — выбор формата (1–3 дня)

- [ ] 30 мин/день TikTok: ниша **iPhone / speaker / audio / photo blur** + 30 мин вне ниши.
- [ ] Составить таблицу 5–10 конкурентов/apps (Speaker Fix, Make Blur, Extract Audio, Cal AI-подобные) и их **повторяющиеся hook-структуры**.
- [ ] Выбрать 1 формат с оценкой Viral / Scalable / Sustainable; зафиксировать hook + demo + CTA в `posts/_formats/`.

### Фаза 1 — MVP под формат (≤2 недели)

- [ ] Убедиться, что app закрывает **ровно обещание формата** (one-tap fix, blur background, MP3 export — без лишних экранов в ролике).
- [ ] Проверить **retention hook**: повод возвращаться (water eject, repeat clean, new photos, new videos).
- [ ] Подготовить 3–5 demo-записей экрана под carousel `experience` / `transformation` слайды.

### Фаза 2 — контент-система (недели 1–4)

- [ ] Заказать/снять **50 hook clips** (реакции, «до/после», frustration calls).
- [ ] Правило: **max 3 uses** на каждый demo clip; вести учёт в таблице.
- [ ] Воскресный batch: съёмка демо + генерация каруселей через `npm run create-post -- <app>`.
- [ ] Портфель: 50% проверенный формат / 25% вариации / 25% эксперименты.
- [ ] 3 поста/день/аккаунт max; разнести по времени; 2+ аккаунта при готовности.

### Фаза 3 — измерение и итерация

- [ ] Трекать **средний view** по аккаунту, не только outliers.
- [ ] A/B hook через carousel title/description (OpenRouter + ручная правка).
- [ ] Связать посты с installs (UTM / App Store analytics) если доступно.

### Фаза 4 — retention и exit optionality

- [ ] Мониторить churn / D7 / D30; усилить onboarding и cancellation value screen в app.
- [ ] Список **стратегических acquirers** (EdTech, productivity, AI writing, health/audio) — 20 компаний.
- [ ] Inbound: единый «creator-style» профиль + consistent stitch; не палить factory look.

### Фаза 5 — автоматизация (только после 100 ручных постов)

- [ ] Не нанимать UGC-фабрику, пока сам не можешь отличить слабый hook от сильного.
- [ ] Тогда: шаблоны в `create-post`, библиотека hooks, возможно creator outreach по Jenni-модели.
